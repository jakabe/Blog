My goal today was to change the Taskboard so that it had some new buttons. Unfortunately I didn't get to that. I did however review how cascading style sheets cascade. This process took longer than I'd like to admit but that does not mean it was without worth.

For my own sake I'll summarize what I went over today.

What is the Cascade?

The word cascade refers to the process which determines the properties of elements in an html file. Occasionally these properties come into conflict. This only occurs when two properties have equal specificity.

What is specificity in the context of the cascading style sheet?

Specificity is the calculated weight of a given property. Web-dev textbooks might define specificity as fallows:

"Specificity is the process that the browser uses to decide which rule applies if there are multiple rules with different selectors".

I am not a fan of this definition. The weight assigned to a propertie's specificity is determined by the types of selectors in that property.

What are selectors in the context of the cascading style sheet?

Selectors are key words that reference tags or attributes of an element in an html file.

How does the browser resolve conflict between element properties?

The browser resolves CSS conflicts by assessing three peices of information
  • Source Order
  • Specificity
  • Importance
Source order refers to the properties position on the cascading style sheet. Properties that come later are given priority. Clearly two properties cannot exist in the same position on the style sheet so the next criteria to be assessed is specificity. The table bellow gives the associated weights for some CSS selectors.

Selector Example Weight
Inline Styles Style="background-color: black;" 1000
ID Selectors id="outer" 100
Class Selectors class = "bright" 10
Element Selectors div 1
A nested paragraph on an html page would probably have a specifity that might look like... html > body > div.grid-container > div.item4 > div.blog > div > div.filterDiv.show.css > article.l > table > tbody ... This would have a very high specificity. Not that you'd ever define a property like that... right?

Lastly there is something called "Importance" which is should never be abused. Yes, any css property will accept the !Important argument. This trumps all. It will also make debugging your CSS far more difficult than usual.
